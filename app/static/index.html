What happens when you tap “Run demo” (and why)
This UI walks through an MVP-grade operator pipeline:

1) signal — ingest a (mock) market/onchain signal.
2) reason — produce a structured “decision memo” (in MVP this is an LLM JSON output).
3) risk gate — hard rules (drawdown, concentration, fee budget). No approval → no action.
4) execute — generate an execution plan (paper-only in this demo).
5) settle — record a (mock) Arc + USDC settlement intent.
6) audit — store an audit record you can fetch later by audit_id.

Goal: show end-to-end discipline, not “a chat that stops at analysis”.
